# Supported Hardware

OpenFlight supports over 2 200 flight simulation peripherals across
five support tiers. This page summarises the tier system, highlights
key devices, and documents known quirks.

> **Auto-generated matrix:** For the full device-by-device table, see
> [`COMPATIBILITY.md`](../../COMPATIBILITY.md) (generated by
> `cargo xtask gen-compat` from manifests in `compat/devices/`).

## Support Tiers

| Tier | Label | Meaning |
|------|-------|---------|
| **0** | Reference | Used for development and CI; full HIL coverage |
| **1** | HIL Validated | Tested on real hardware in the lab |
| **2** | Automated Tests | Simulated/unit-tested; no physical device needed |
| **3** | Best Effort | Manifest exists with VID/PID and capabilities |
| **4** | Community | User-contributed; minimal validation |
| **5** | Experimental | Partial or unverified support |

## Force Feedback Devices

FFB support is available for select devices through `flight-ffb`.
All FFB operations are governed by safety interlocks
([ADR-009](../explanation/adr/009-safety-interlock-design.md)):

| Device | FFB Type | Status |
|--------|----------|--------|
| Saitek X65F | HID FFB | Tier 2 |
| Brunner CLS-E | Proprietary | Tier 3 |
| Microsoft SideWinder FFB2 | HID FFB | Tier 3 |

### FFB Safety

- **Envelope enforcement**: Forces are clamped to safe maximums
- **Ramp-to-zero**: On fault detection, all forces ramp to zero
  within a bounded time
- **Watchdog timeout**: If the RT spine misses ticks, FFB
  disengages automatically

> Run `cargo test -p flight-ffb safety` and
> `cargo test -p flight-ffb envelope` before modifying FFB code.

## Known Quirks

Some devices have hardware-specific behaviours documented in their
manifests (the `quirks` section of each YAML file):

| Device | Quirk | Description |
|--------|-------|-------------|
| X52 Pro | `MFD_BRIGHTNESS` | MFD brightness uses USB control transfer (report 0xB1), not standard HID output |
| X52 Pro | `LED_RGB` | Six RGB LEDs via HID output reports with 3-bit R/G/B |
| T.Flight HOTAS One | `AXIS_JITTER` | Potentiometers can produce ±2 LSB noise; use EMA filter |
| Some VKB sticks | `FIRMWARE_VERSIONS` | Report layout differs between firmware versions |

## Adding a New Device

If your device is not listed, you can contribute support:

1. Create a device manifest in `compat/devices/<vendor>/`
2. Optionally write a HOTAS crate with HID report parsing
3. See [Adding a Device](../how-to/adding-a-device.md) for the full
   guide

## Regenerating the Matrix

The full compatibility matrix in `COMPATIBILITY.md` is generated from
all YAML manifests:

```bash
cargo xtask gen-compat
```
